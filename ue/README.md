# Acro Sim ‚Üî Unreal Engine bridge

Drive Unreal rendering from the Dart `acro_space_simulator`. The Dart sim stays
authoritative; UE is a client that sends player intent and renders the world the
sim streams back. One FlatBuffers schema (`../wire/sim.fbs`) is the only
contract ‚Äî neither side hand-writes marshalling.

## 1. Run the sim server (Dart side)

```
dart run bin/sim_server.dart            # 127.0.0.1:5800, 20 Hz
dart run bin/sim_server.dart 5800 30    # custom port / tick rate
```

It streams a `WorldFrame` every tick and applies the `CommandFrame`s clients
send. Pure Dart VM ‚Äî no Flutter engine needed.

## 2. Add the module (UE side)

Copy `AcroSimBridge/` into your project (either as `Source/AcroSimBridge` or a
plugin under `Plugins/AcroSimBridge`). It already contains:

- `Source/AcroSimBridge/` ‚Äî `USpaceSimSubsystem` (.h/.cpp) + `AcroSimBridge.Build.cs`
- `Wire/sim_generated.h` ‚Äî the FlatBuffers C++ contract (generated by `flatc`)
- `ThirdParty/flatbuffers/include/` ‚Äî the header-only FlatBuffers runtime (vendored)

Add `"AcroSimBridge"` to your `.uproject`/`.uplugin` modules (or paste the
subsystem + the two include paths from `Build.cs` into your existing game
module). Regenerate project files, build.

## Quick render: drop in `ASpaceSimRenderer`

Fastest path ‚Äî no per-frame code. Place an **AcroSimRenderer** actor in the level
and set its **Asset Table** (a DataTable whose row struct is `FAcroAssetRow`). Row
name = the type-key the sim sends:

- a **body id** (`kerbin`, `mun`, `earth`) ‚Üí planet mesh (a unit-radius sphere; the
  renderer scales it to the body's real radius ‚Äî set **Body Mesh Unit Radius Cm**
  to your sphere's radius, `50` for UE's default sphere),
- a **part name** (`LV-T45`, ‚Ä¶) ‚Üí part mesh,
- a **building spec type** (`refinery`, `hab`, `solar`, ‚Ä¶) ‚Üí building mesh.

Set a **Fallback Mesh** (debug cube) so missing keys stay visible. With **Auto
Connect** on, it connects on BeginPlay and each frame: find-or-spawns body/craft
actors, composes craft from `Parts`, HISM-instances buildings under the rotating
body actor, and prunes anything that leaves the stream (staging drops part meshes;
despawns vanish). Enable **Report Terrain** (+ a trace channel) to raycast your
landscape and reconcile heights.

Author the DataTable, press Play. For bespoke control (custom actor classes,
effects, UI) bind `On World Updated` yourself ‚Äî see below.

## 3. Use it (manual / Blueprint)

`USpaceSimSubsystem` is a `UGameInstanceSubsystem`, so `GetGameInstance()->GetSubsystem<USpaceSimSubsystem>()` from anywhere. From your level/pawn `BeginPlay`:

```cpp
auto* Sim = GetGameInstance()->GetSubsystem<USpaceSimSubsystem>();
Sim->FocusBodyId = TEXT("kerbin");   // becomes the UE world origin
Sim->Connect(TEXT("127.0.0.1"), 5800);
```

Each frame, read the rebased, UE-space world and drive your actors (lerp toward
the new transforms rather than snapping):

```cpp
for (const FSimBody& B : Sim->GetBodies())
{
    AActor* Planet = FindOrSpawnBodyActor(B.Id);  // keep a handle per body id
    Planet->SetActorLocationAndRotation(B.Position, B.Orientation);
    // scale a unit sphere to B.RadiusCm
}

for (const FSimVessel& V : Sim->GetVessels())
{
    AActor* Craft = FindOrSpawnVesselActor(V.Id);
    Craft->SetActorLocationAndRotation(V.Position, V.Attitude); // cm + UE handedness
    // Compose the craft from its parts (rebuild only when the part set changes):
    for (const FSimPart& P : V.Parts)
    {
        UStaticMeshComponent* Mesh = FindOrAddPartMesh(Craft, P.Id, MeshFor(P.Type));
        Mesh->SetRelativeLocation(P.LocalOffset); // craft-local; inherits V.Attitude
    }
}

for (const FSimBuilding& Bld : Sim->GetBuildings())
{
    // Body-FIXED: parent under the body actor so it spins with the planet.
    AActor* BodyActor = FindOrSpawnBodyActor(Bld.Body);
    UInstancedStaticMeshComponent* Ism = HismFor(BodyActor, Bld.Type); // 1 HISM per type
    FTransform Local(Bld.LocalOrientation, Bld.LocalPosition);
    AddOrUpdateInstance(Ism, Bld.Colony + TEXT("/") + Bld.Id, Local); // ids are colony-scoped

    // Snap to your landscape and tell the sim, so both agree (bidirectional):
    const float HeightM = RaycastTerrainHeightMetres(BodyActor, Bld.Lat, Bld.Lon);
    Sim->SubmitReportTerrainHeight(Bld.Body, Bld.Lat, Bld.Lon, HeightM);
}
```

Parent buildings (and landed craft) under the body actor and let it carry the
spin ‚Äî never recompute planet rotation per object. Use a HISM per building type
so the engine instances/culls/LODs thousands of them. After you report a height,
the next frame's `Bld.LocalPosition` already includes it, so the sim's placement
and your landscape converge.

Send intent:

```cpp
Sim->SubmitThrottle(TEXT("demo-1"), 1.0f);
Sim->SubmitAttitude(TEXT("demo-1"), FVector(0, 1, 0)); // forward axis in SIM coords
Sim->SubmitSeparateStage(TEXT("demo-1"));
Sim->SubmitPlaceBuilding(TEXT("colony-1"), TEXT("hab"), 2, 3);
```

> `SubmitPlaceBuilding` is wired across the contract, but on the Dart side
> `ApplyCommands` currently handles `PlaceBuildingCommand` as a no-op (reserved).
> It travels end-to-end with no gameplay effect until that case is implemented.

## Blueprint usage

Everything is exposed to Blueprint ‚Äî no C++ needed to drive it:

- **Get the subsystem:** `Get Game Instance Subsystem` ‚Üí class `SpaceSimSubsystem`.
- **Connect / commands:** `Connect`, `Disconnect`, `IsConnected`, `Submit Throttle`,
  `Submit Attitude`, `Submit Separate Stage`, `Submit Place Building`,
  `Submit Report Terrain Height` are all BlueprintCallable. `Focus Body Id` /
  `Player Id` are editable variables.
- **Render on an event, not Tick:** bind the **`On World Updated`** event (fires
  after each frame). In the handler, call `Get Vessels` / `Get Bodies` /
  `Get Buildings` (cache to a local array, then `For Each`).
- **Transforms:** the `FSim*` structs expose every field, but use the pure nodes
  `Vessel Transform` / `Body Transform` / `Building Local Transform` to get a
  ready `Transform` pin (cleaner than the raw `FQuat` fields) for
  `Set Actor Transform` / `Add Instance`.
- **Telemetry / gauges:** each `FSimVessel` carries `Mass Kg`, `Crew`,
  `Resources` (type / amount / capacity, summed by type), and `Max Temp K` /
  `Temp Limit K` ‚Äî wire straight to HUD gauges or a reentry-glow material param.
- **Events / FX:** bind **`On Sim Event`** ‚Äî fires once per discrete sim event
  (staging, impact, destruction, crew lost, structural failure, science, ‚Ä¶).
  Switch on `Kind`; `Subject` is the asset id (look it up in `Get Vessels` for
  the spawn point, e.g. an explosion at an `Impact`), `Magnitude`/`Info` carry
  the payload (impact speed, overheat temperature, crew-loss cause).

A pure-Blueprint consumer: on `On World Updated` ‚Üí `Get Bodies` ‚Üí For Each ‚Üí
find-or-spawn a body actor (keep a `Map<String, Actor>` variable) ‚Üí
`Set Actor Transform (Body Transform)`; same shape for vessels (+ `Parts`) and
buildings (parent under the body actor, `Add Instance` to a per-type HISM with
`Building Local Transform`). You still author the `Type ‚Üí Mesh` mapping and the
actor/instance bookkeeping in Blueprint.

## Coordinate conventions (important)

| | Sim (Dart) | Unreal |
|---|---|---|
| Handedness | right-handed | left-handed |
| Up axis | Z | Z |
| Units | metres, m/s | centimetres |
| Quaternion | Hamilton, **scalar-first** (w,x,y,z) | FQuat **scalar-last** (x,y,z,w) |

`USpaceSimSubsystem` converts on ingest (`SimToUEPosition` / `SimToUEQuat`):
negate Y to flip handedness, √ó100 for cm, and remap the quaternion to
`(w, ‚àíx, y, ‚àíz)`. **Verify against one known craft pose** the first time you run
it ‚Äî if meshes face backwards, flip the sign convention in those two helpers
(one place).

### Floating origin

Sim positions are huge (body positions are system-root-relative, AU-scale).
The subsystem rebases everything so `FocusBodyId` sits at the UE world origin:
vessel world = `bodyPos[vessel.body] + vessel.localPos`, then minus
`bodyPos[FocusBodyId]`. Single-precision `float` stays accurate near the focus.
For very large scenes, move `FocusBodyId` (or layer UE's World Origin Rebasing /
Large World Coordinates on top).

## Threading & pacing

The sim ticks at its own fixed rate; UE renders at its own. The subsystem reads
the **latest** complete frame each UE tick and you interpolate toward it. For
production, move the socket recv to a worker thread (FRunnable) and double-buffer
‚Äî the current single-threaded `Tick` recv is fine to start.

`WorldFrame.fingerprint` rides along as a cheap desync check if you ever run a
local predicted copy.

Parts and buildings ride in **every** `WorldFrame` (the frame is stateless ‚Äî
latest wins). Cache vessel actors, part meshes, and building instances by id and
only rebuild when the set actually changes. If cities grow large enough that
re-sending every building per tick costs too much, split buildings into a
separate lower-rate placement stream ‚Äî the schema and `FSimBuilding` don't
change, only how often you send it.

## Regenerating the contract

After editing `../wire/sim.fbs`, regenerate both sides:

```
flatc --dart            -o ../lib/adapters/wire   ../wire/sim.fbs
flatc --cpp --scoped-enums -o Wire                ../wire/sim.fbs
```

Keep the union cases in `sim.fbs`, `ApplyCommands` (Dart), and the
`flatbuffer_codec.dart` switch in lockstep.

## Faster transport later

TCP loopback is the first cut (least code, `FSocket` on the UE side). For lower
latency, swap the transport for a shared-memory ring buffer holding the latest
`WorldFrame` ‚Äî the schema, codec, and subsystem logic don't change, only the
bytes' delivery path.
